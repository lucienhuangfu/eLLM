## Git Commit Message Rules

### Language Requirement
- **Must use English** for commit messages
- All content (including title and description) must be in English

### Format Guidelines
- Use a concise title (max 50 characters)
- Add a description after a blank line if needed
- Start with a verb, e.g., "fix", "add", "update", "refactor", "remove", etc.

### Examples
- fix: user login error
- add: new API endpoint
- update: dependency versions
- refactor: auth module code
